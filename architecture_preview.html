What Each New Layer Actually Does

🟢 ML Layer — Numerical Intelligence

  • Reads directly from fact_pricing_snapshots + fact_daily_weather
  • Trains offline on historical partitions, scores online per event
  • Outputs: numerical scores, forecasts, cluster labels, anomaly flags
  • Writes predictions back to a new ml_predictions table in Postgres
  • Feeds alerts.price_drops with enriched context scores
  • Does NOT speak in English — outputs structured, typed data only

🟣 LLM Layer — Language Intelligence

  • Reads from ml_predictions table — never raw sensor data
  • Also reads from Star Schema directly for Text-to-SQL queries
  • Subscribes to alerts.price_drops for real-time narration
  • Converts opaque numbers into human-legible narratives
  • Cannot learn or update from new data — ML does the learning
  • Powers chatbot / dashboard copilot / daily digest email
⚖️

Core Design Principle: ML handles what is happening — pattern detection, forecasting, anomaly scoring, correlation quantification. LLM handles why it matters and what to do — explanation, narrative, recommendation, and natural language access. The LLM is a translator that sits above the ML layer, not a replacement for it. Raw data never reaches the LLM directly; it always flows through the structured intelligence of the Star Schema and ML outputs first.
